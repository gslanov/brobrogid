### 1.4 Иерархия экранов

**L1 — Табы (всегда доступны):**
- `/` (Главная)
- `/map` (Карта)
- `/tours` (Туры)
- `/profile` (Профиль)

**L2 — Один тап от таба:**
- `/search` ← Главная (тап по SearchBar)
- `/tours/:id` ← Туры
- `/emergency` ← Главная (Quick Access) + глобальный SOS
- `/poi/:id` ← Карта, Поиск, Главная
- `/profile/saved` ← Профиль

**L3 — Два тапа от таба:**
- `/poi/:id/menu` ← Детали POI
- `/tours/guide/:id` ← Детали тура
- `/cart` ← Меню ресторана
- `/profile/subscription` ← Профиль

### 2.3 Services Grid (вместо Quick Access)

**Было:** 4 кнопки 48px с emoji (🚨🍽️🗺️🚌), текст 11px

**Стало:** сетка 2x3 или 2x2 с SVG-иконками, высота ячейки 70-80px, текст 13-14px

```
Ряд 1:  Где поесть  |  Что посмотреть  |  Туры
Ряд 2:  Карта        |  Транспорт       |  SOS
```

Или минимум (2x2):
```
Где поесть  |  Туры
Транспорт   |  SOS
```

- Формулировки action-oriented: не "Food", а "Где поесть"
- Emergency (SOS) — не первая кнопка, но присутствует
- Каждая ячейка минимум 64px высоты, текст минимум 13px

### 2.4 Что убрать с главной

- Карусель "Tours" → туры теперь отдельный таб

- CategoryBottomSheet → не нужен при 6-7 чипах
- Emergency из первой позиции Quick Access → SOS-кнопка глобальная

### 2.5 Глобальная SOS-кнопка

Доступна с ЛЮБОГО экрана. Варианты реализации:

### Общий вердикт
Приложение — "красивый прототип" с крепким техническим фундаментом (MapLibre, Fuse.js, IndexedDB, lazy loading, PWA). Проблемы чисто UX-уровня: перегруженная главная, 50% кнопок — пустышки, данные из модели не доходят до UI, inconsistency между экранами, нет завершённых коммерческих flow (бронирование, заказ еды).

**Для превращения из прототипа в продукт нужно:**
1. Починить все мёртвые кнопки (P0, ~3 часа)
2. Реализовать минимальные flow бронирования и заказа (P1, ~10 часов)
3. Показать скрытые данные из модели (P1, ~3 часа)
4. Заменить emoji на иконки (P1, ~3 часа)
5. Реструктуризировать навигацию (P1-P2, ~5 часов)

**Итого на P0+P1: ~25-30 часов работы.**
