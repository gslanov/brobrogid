# i18n в app.brobrogid.ru

## Stack

- `react-i18next` + `i18next`
- Backend: статические JSON в `src/i18n/locales/{ru,en}.json`
- Detector: localStorage (`brobrogid_lang`) → navigator → fallback `ru`

## Конфиг

`src/i18n/index.ts`:

```ts
i18n.use(initReactI18next).init({
  fallbackLng: 'ru',
  supportedLngs: ['ru', 'en'],
  defaultNS: 'common',
  resources: {
    ru: { common: ruCommon, admin: ruAdmin, ... },
    en: { common: enCommon, admin: enAdmin, ... }
  }
})
```

## Namespaces

| NS | Кол-во ключей | Где |
|---|---|---|
| `common` | ~400 | UI elements, buttons, errors |
| `admin` | ~230 | Admin panel UI |
| `categories` | ~30 | POI categories |
| `errors` | ~50 | Error messages |

## Контент vs UI

**Важное разделение:**

1. **UI strings** (кнопки, лейблы) → `useTranslation()` + ключи в JSON
   ```tsx
   const { t } = useTranslation()
   <button>{t('common.save')}</button>
   ```

2. **Контент данных** (название POI, описание тура) → `LocalizedText` тип в самих данных:
   ```ts
   interface POI {
     name: { ru: string; en: string }
     description: { ru: string; en: string }
   }
   ```
   Рендеринг через хук:
   ```tsx
   const lang = useLanguage()
   <h1>{poi.name[lang]}</h1>
   ```

## useLanguage hook

```ts
export function useLanguage(): 'ru' | 'en' {
  const { i18n } = useTranslation()
  return (i18n.language?.startsWith('en') ? 'en' : 'ru')
}
```

Используется везде где нужно достать локализованное поле из данных.

## LocalizedText helper

```ts
export function localizedText(
  text: { ru: string; en: string } | string | undefined,
  lang: 'ru' | 'en'
): string {
  if (!text) return ''
  if (typeof text === 'string') return text
  return text[lang] || text.ru || ''
}
```

Защищает от случаев когда EN перевод отсутствует — fallback на RU.

## Переключатель языка

`LanguageToggle` компонент в header:
```tsx
function toggleLang() {
  const next = i18n.language === 'ru' ? 'en' : 'ru'
  i18n.changeLanguage(next)
  localStorage.setItem('brobrogid_lang', next)
}
```

## Что НЕ локализовано

- POI с пустым `name.en` → fallback на ru (через `localizedText`)
- Часть `description.en` отсутствует у новых POI — TODO для контент-агента
- Admin форма редактирует оба языка одновременно (`LocalizedInput` компонент)

## Связь с SEO

`<html lang="ru">` — корневой язык страницы. При переключении на EN — `lang` НЕ меняется prerendered HTML (только runtime через Helmet).

После Pool 4 / SEO launch: prerender нужно делать дважды (ru и en версии URL) — `/poi/slug` и `/en/poi/slug`. Сейчас не реализовано.
